Assignment of the Week: BeEF Hook via Reflected XSS

Week #: 01 Difficulty: Beginner Estimated Time: 30–60 min Category: Web / Client-Side

Scenario

You are assessing an internal training web application. You suspect user input is reflected without sanitization. Your goal is to validate whether a client-side compromise is possible in a controlled lab.

Lab only. Authorized testing only. Do not deploy DVWA or BeEF on public systems.

Objective

  • Confirm reflected input behavior.
  • Demonstrate JavaScript execution.
  • Hook a browser using BeEF.
  • Demonstrate one safe post-exploitation command (e.g., Alert Dialog).

Environment

  • Target: DVWA on Ubuntu Server (no GUI) or RHEL
  • Attacker: Kali Linux with BeEF
  • Network: Same LAN / Host-only / NAT (no Internet exposure)

Rules of Engagement

  • No denial-of-service.
  • No persistence / no changing server files.
  • Capture evidence using screenshots and short notes.

Tasks

  1. Navigate to the vulnerable page in DVWA.
  2. Prove reflection (text appears in output).
  3. Prove JavaScript execution (use a safe test).
  4. Start BeEF and identify the Hook URL.
  5. Inject the hook and confirm the browser appears under “Hooked Browsers”.
  6. Run one safe BeEF command (e.g., Alert Dialog) and capture evidence.

Flags

FLAG{reflected_input_confirmed}
FLAG{javascript_execution_confirmed}
FLAG{browser_hooked_successfully}
FLAG{client_side_control_achieved}

Deliverables

  • Screenshot: DVWA page showing reflected input.
  • Screenshot: Evidence of JavaScript execution (or a safe indicator).
  • Screenshot: BeEF panel showing hooked browser.
  • Screenshot: BeEF command result (e.g., Alert Dialog popup).
  • Short write-up (5–10 sentences): What happened and why it worked.
  • Mitigation (2–3 bullets): How to prevent this in real apps.

Notes / Observations

  • DVWA IP: ____________
  • Kali IP: ____________
  • DVWA Security Level: Low / Medium / High
  • What worked / what failed: ____________________________

Instructor / Self-Check

  • ✅ I can explain the difference between server compromise and browser compromise.
  • ✅ I can describe at least one mitigation (CSP / output encoding / HttpOnly cookies).
  • ✅ I captured evidence and wrote a short summary.
